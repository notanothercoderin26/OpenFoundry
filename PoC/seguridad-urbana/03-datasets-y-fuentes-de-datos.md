# 03 — Datasets and data sources (~80 GB total, public + legal)

> **Golden rule:** all PoC data is **public, legal, and licensed for use under the terms each provider publishes**. Zero internal police records, zero classified data, zero non-public PII. Every source has attribution and a license note shown in the app footer.
>
> **Volume note — read this on stage.** This PoC targets **~80 GB raw**, not TB-scale. Urban public-safety data does not naturally exist at TB volume: the city of Barcelona publishes ~1 M incidents and ~50 k accidents across five years and that is the real ceiling. We **deliberately do not inflate** the headline number — the value of this PoC is in the **decision loop** (forecast → mark zone → close zone → after-action review), not in storage size. The ~50–70 GB of synthetic AVL traces exist to exercise the patrol-coverage workflow and the markings/PII story, and are openly labeled `SYNTH` in the ontology.
>
> The demo-day subset is declared in [`assets/sample-data-manifest.yaml`](assets/sample-data-manifest.yaml) (created at execution time) and ingested via [`infra/scripts/poc-seguridad-urbana/fetch-sample-data.sh`](../../infra/scripts/poc-seguridad-urbana/fetch-sample-data.sh) per the runbook at [`infra/runbooks/poc-seguridad-urbana-ingest.md`](../../infra/runbooks/poc-seguridad-urbana-ingest.md).

## 1️⃣ Open Data BCN — `incidents-gestionats-gub` (the workhorse)

**What it is:** the catalogue of incidents managed by **Guàrdia Urbana de Barcelona** between 2020 and 2024. Each row is a single dispatched intervention with typology, district, neighborhood, street, and anonymized coordinates.

**Why it's the workhorse:** real, public, attribution-friendly municipal-police data with five years of history at neighborhood resolution. It is the spine of `Incident` in the ontology and the training target for the load-forecast model.

### Schema highlights (subset relevant to the ontology)

| field | type | notes |
|---|---|---|
| `numero_expedient` | string | unique incident ID |
| `data_creacio` | timestamp | used by `shift-binner` to assign MAÑANA/TARDE/NOCHE/MADRUGADA |
| `nom_districte` | string | one of the 10 districts |
| `nom_barri` | string | one of the 73 barrios |
| `tipus_incident` | string | top-level typology |
| `subtipus_incident` | string | sub-typology |
| `tipologia` | string | normalized typology label |
| `via` | string | street (no number — already anonymized at source) |
| `latitud_amb_soroll` | float | latitude with noise added at source for anonymization |
| `longitud_amb_soroll` | float | longitude with noise added at source for anonymization |

### Download

```bash
mkdir -p /data/raw/bcn/incidents
for YEAR in 2020 2021 2022 2023 2024; do
  curl -fsSL \
    "https://opendata-ajuntament.barcelona.cat/data/dataset/incidents-gestionats-gub/resource/incidents-${YEAR}.csv" \
    -o "/data/raw/bcn/incidents/incidents-${YEAR}.csv"
done
aws s3 sync /data/raw/bcn/incidents/ s3://bcn-publicsafety-poc/raw/bcn/incidents/
```

> Dataset landing page: <https://opendata-ajuntament.barcelona.cat/data/es/dataset/incidents-gestionats-gub>

### Mandatory attribution
> *"Source: Open Data BCN — Ajuntament de Barcelona, Guàrdia Urbana. CC-BY 4.0."* shown in the dashboard footer.

## 2️⃣ Open Data BCN — `accidents-gu-bcn`

## 3️⃣ Open Data BCN — neighborhoods + districts geometry

**What it is:** the official **administrative division** of Barcelona — 73 *barris* and 10 *districtes* — as GeoJSON polygons.

**Why we use it:** these polygons populate the `Neighborhood` and `District` object types and are the spatial keys for joining incidents, accidents, weather, census, and AVL traces. Every forecast cell is `(barrio, shift, date)`.

### Download

```bash
mkdir -p /data/raw/bcn/geom
curl -fsSL \
  "https://opendata-ajuntament.barcelona.cat/data/dataset/divisio-administrativa-barris/resource/barris.geojson" \
  -o /data/raw/bcn/geom/barris.geojson
curl -fsSL \
  "https://opendata-ajuntament.barcelona.cat/data/dataset/divisio-administrativa-districtes/resource/districtes.geojson" \
  -o /data/raw/bcn/geom/districtes.geojson
aws s3 sync /data/raw/bcn/geom/ s3://bcn-publicsafety-poc/raw/bcn/geom/
```

## 4️⃣ AEMET OpenData — hourly weather observations

**What it is:** the Spanish State Meteorological Agency's open data portal. We use the **hourly observation** endpoints for the two Barcelona stations:
- **El Prat (Aeropuerto)** — station code `0076` — coastal / airport, the city-wide reference.
- **Fabra (Observatori)** — station code `0200E` — Tibidabo, the historical observatory.

**Why we use it:** weather (rain Y/N, temperature bin) is a real feature in the load-forecast model and the panel in UC-2 ("incidents during rain vs dry hours"). AEMET is free with a self-service API key.

### Example API call

```bash
# 1. Request an API key at https://opendata.aemet.es/centrodedescargas/altaUsuario
export AEMET_API_KEY=...

# 2. Two-step retrieval: AEMET returns a signed URL to the actual payload
META=$(curl -fsSL \
  "https://opendata.aemet.es/opendata/api/valores/climatologicos/diarios/datos/fechaini/2020-01-01T00:00:00UTC/fechafin/2024-12-31T23:00:00UTC/estacion/0076" \
  -H "api_key: ${AEMET_API_KEY}")
DATA_URL=$(echo "$META" | jq -r .datos)
curl -fsSL "$DATA_URL" -o /data/raw/aemet/el-prat-2020-2024.json

# Repeat for station 0200E (Fabra)
aws s3 sync /data/raw/aemet/ s3://bcn-publicsafety-poc/raw/aemet/
```

> Portal: <https://opendata.aemet.es/>

### Mandatory attribution
> *"Source: AEMET — Agencia Estatal de Meteorología (OpenData)."* shown in the dashboard footer.

## 5️⃣ INE — census sections of Barcelona

**What it is:** the Spanish **National Statistics Institute** publishes the census-section geometries and per-section population / age-bracket / household counts. Barcelona has ~1.1 k census sections nested inside the 73 barrios.

**Why we use it:** the `CensusTract` object type provides the **population-density** feature for the forecast model and the **PII-RESIDENT-gated** demographic columns (age bracket, household composition) that exercise the marking story in UC-7. Sub-neighborhood granularity is intentionally locked behind the `PII-RESIDENT` marking.

### Download

```bash
mkdir -p /data/raw/ine/census
curl -fsSL \
  "https://www.ine.es/jaxi/files/_px/es/csv_bdsc/t20/e245/p07/a2024/l0/0008.csv" \
  -o /data/raw/ine/census/bcn-sections-2024.csv
aws s3 sync /data/raw/ine/census/ s3://bcn-publicsafety-poc/raw/ine/
```

> Portal: <https://www.ine.es/> (census-section tables under "Cifras de población y Censos demográficos").

### Mandatory attribution
> *"Source: INE — Instituto Nacional de Estadística (census sections, Barcelona)."*

## 6️⃣ Synthetic — patrol AVL traces (`SYNTH`)

> Needed because no public dataset gives us **GPS traces of patrol units** at operational cadence. We generate them deterministically to (a) demonstrate the patrol-coverage workflow in the Workshop module and (b) exercise the markings / PII story (patrol-officer IDs are coarsened for users without the right marking) without using any real personnel data.

### Synthetic schema (`patrol_avl.parquet`)

| column | type | distribution |
|---|---|---|
| `patrol_id` | string | one of 60 demo patrol units (3 per district + 30 city-wide) |
| `unit_type` | enum | `FOOT` / `CAR` / `MOTO` / `BIKE` |
| `ts` | timestamp | 5-second cadence, 2024-01-01 → 2024-12-31 |
| `lat`, `lon` | float | constrained to the 73 barrio polygons via random walk |
| `barrio_id` | string | derived from `(lat, lon)` against the BCN geometry |
| `shift` | enum | `MAÑANA` / `TARDE` / `NOCHE` / `MADRUGADA` |
| `event_marker` | nullable enum | sparse events: `RESPONDING` / `ON_SCENE` / `IDLE` |

### Generator (deterministic, seed = 42)

```bash
python tools/poc-seguridad-urbana/generate_patrol_avl.py \
  --geom        s3://bcn-publicsafety-poc/raw/bcn/geom/barris.geojson \
  --output      s3://bcn-publicsafety-poc/raw/synthetic/patrol_avl/ \
  --start       2024-01-01 --end 2024-12-31 \
  --cadence-seconds 5 --units 60 --seed 42
```

> **Reproducibility:** the generator must be deterministic so rehearsals match the demo exactly. Every record carries `source = "SYNTH"` so the demo can openly disclose this on stage.

## 7️⃣ Synthetic — historical CAD-style dispatch records (`SYNTH`)

> Used in **UC-5** (live alert → patrol re-assignment). Real CAD records are not public; we generate a deterministic stream that joins to real `Incident` IDs from source 1 so the demo's live alert flow has plausible call-handling data to act on.

### Synthetic schema (`cad_dispatch.parquet`)

| column | type | notes |
|---|---|---|
| `dispatch_id` | string | UUID, deterministic |
| `incident_id` | string | FK to `numero_expedient` from source 1 (when a match exists) |
| `received_at` | timestamp | offset uniformly within the hour of the incident |
| `assigned_patrol_id` | string | FK to `patrol_avl.patrol_id` |
| `response_seconds` | int | Gamma(2, 90) — most calls answered < 5 min |
| `outcome` | enum | `RESOLVED` / `ESCALATED` / `STOOD_DOWN` |
| `narrative_snippet` | string | templated, used as retrieval corpus for AIP |

### Generator (deterministic, seed = 42)

```bash
python tools/poc-seguridad-urbana/generate_cad_dispatch.py \
  --incidents s3://bcn-publicsafety-poc/raw/bcn/incidents/ \
  --patrols   s3://bcn-publicsafety-poc/raw/synthetic/patrol_avl/ \
  --output    s3://bcn-publicsafety-poc/raw/synthetic/cad_dispatch/ \
  --seed 42
```

## 🛂 Markings applied at ingest

This is the headline differentiator for this PoC. Each raw dataset is tagged with one or more markings on ingestion; downstream curated datasets inherit them unless a transform explicitly redacts.

| Dataset | Markings | Effect |
|---|---|---|
| `raw.bcn.incidents` | `PUBLIC-OPENDATA`, `ANALYST-CORE` | Jordi sees raw detail (typology + street); Anna sees aggregated counts per barrio. |
| `raw.bcn.accidents` | `PUBLIC-OPENDATA`, `ANALYST-CORE` | Same pattern. |
| `raw.bcn.geom`      | `PUBLIC-OPENDATA` | Visible to both. |
| `raw.aemet.*`       | `PUBLIC-OPENDATA` | Visible to both. |
| `raw.ine.census`    | `PUBLIC-OPENDATA`, `PII-RESIDENT` | Coarse counts public; age/household columns gated behind `PII-RESIDENT`. Neither Anna nor Jordi has `PII-RESIDENT` in the default demo — the AIP refusal in UC-7 fires on this. |
| `raw.synthetic.patrol_avl` | `OPS-CORE` | Anna sees raw patrol movement; Jordi sees barrio-aggregated coverage only. |
| `raw.synthetic.cad_dispatch` | `OPS-CORE`, `ANALYST-CORE` | Anna sees dispatch + outcome; Jordi sees narrative snippets used by AIP retrieval. |

> The marking enforcement is provided by `authorization-policy-service` and applied by `dataset-versioning-service` on read. See [`10-seguridad-y-gobierno.md`](10-seguridad-y-gobierno.md) for the policy definitions.

## ✅ Concrete actions (when the PoC is executed)

1. Create accounts and tokens: AEMET API key (self-service, 24 h email validation), AWS / MinIO bucket `bcn-publicsafety-poc`. Open Data BCN and INE require no account.
2. Develop scripts under `tools/poc-seguridad-urbana/`:
   - `download_bcn_incidents.sh`, `download_bcn_accidents.sh`, `download_bcn_geom.sh`
   - `download_aemet_hourly.py`
   - `download_ine_census.sh`
   - `generate_patrol_avl.py`, `generate_cad_dispatch.py` (both seeded `42`)
3. Kick off downloads at least 48 h ahead — AEMET's two-step signed-URL flow is rate-limited; everything else is single-shot.
4. Verify total volume: `aws s3 ls s3://bcn-publicsafety-poc --recursive --summarize --human-readable | tail -3` and confirm it is **≈ 80 GB**, not TB-scale — the volume itself is part of the honest narrative.
5. Apply markings at ingest (the policy IDs and assignment calls are in [`10-seguridad-y-gobierno.md`](10-seguridad-y-gobierno.md)); double-check that `raw.ine.census` correctly gates `PII-RESIDENT` columns.
6. Load the license footer (all six attribution strings above) into the UI before the first rehearsal.
