# CLAUDE.md — sdks/

OpenFoundry's official client SDKs. **Most of this directory is
generated code; some packages are hand-written.** Read the table below
before editing anything.

## What is generated vs hand-written

| Path | Status | Source of truth | Regenerate with |
|---|---|---|---|
| `sdks/typescript/openfoundry-sdk/src/*.ts` | **Generated** | `apps/web/public/generated/openapi/openfoundry.json` | `go run ./tools/of-cli docs generate-sdk-typescript ...` |
| `sdks/python/openfoundry-sdk/openfoundry_sdk/*.py` | **Generated** | `apps/web/public/generated/openapi/openfoundry.json` | `go run ./tools/of-cli docs generate-sdk-python ...` |
| `sdks/java/openfoundry-sdk/src/main/java/com/openfoundry/sdk/*.java` | **Generated** | `apps/web/public/generated/openapi/openfoundry.json` | `go run ./tools/of-cli docs generate-sdk-java ...` |
| `sdks/python/openfoundry_transforms/openfoundry_transforms/*.py` | **Hand-written** | none — Foundry-doc-shaped naming | edit directly |
| `sdks/python/openfoundry-sdk/examples/*.py` | **Hand-written** examples | none | edit directly |
| `sdks/*/openfoundry-sdk/README.md`, `pyproject.toml`, `package.json`, `pom.xml` | **Hand-written** | none | edit directly |

Every generated file carries an in-file marker:

```
// This file is generated by `go run ./tools/of-cli docs generate-sdk-<lang>`.
// Do not edit manually.
```

If the marker is missing, the file is hand-written.

## Regenerating all SDKs

```sh
make gen   # runs gen-proto + gen-sqlc + contracts-gen
make contracts-check   # verifies OpenAPI + per-SDK drift
```

CI gates the drift check via [`.github/workflows/proto-check.yml`](../.github/workflows/proto-check.yml).

## When to edit hand-written code

- **`openfoundry_transforms/`** — this is OpenFoundry's port of
  Foundry's `transforms.api` / `transforms.mediasets` /
  `transforms.mediasets.virtual_tables` Python surface. Edit the
  source files directly. The naming intentionally mirrors upstream
  Foundry tutorials so users can copy-paste.
- **Examples (`examples/*.py`)** — change freely; they're documentation.
- **Each SDK's `README.md`, `pyproject.toml`, `package.json`, `pom.xml`** —
  package metadata + usage docs; hand-edit.

## Don't

- **Don't edit a file with the "Do not edit manually" header.** Your
  change will be erased on the next `make gen`. If you need a behavior
  change, edit either the `.proto` files (under `proto/`), the OpenAPI
  source (under `apps/web/public/generated/openapi/`), or the
  generator in [`tools/of-cli/docs.go`](../tools/of-cli/docs.go).
- **Don't add a hand-written file under `openfoundry-sdk/<lang>/...`** —
  that path is owned by the generator. Put hand-written helpers in a
  sibling package (the way `openfoundry_transforms/` does).
- **Don't bump the `OPENFOUNDRY_SDK_VERSION` constant in
  `index.ts`** by hand; it's generated from the version pin in the
  generator script.
